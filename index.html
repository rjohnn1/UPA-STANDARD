Hosted Schema

Lightweight Vendor Whitelist

For enterprise deployments without full ERP integration, UPA-1.2 specifies a lightweight alternative: a human-attested JSON configuration that the Fiduciary Translation Engine validates against — identical enforcement characteristics, simplified data source.

Reference this schema in any UPA-compliant whitelist configuration file using the $schema field. Any compliant JSON Schema validator will enforce the structure.

GET
github.com/rjohnn1/UPA-STANDARD/v1/lightweight-whitelist.json
my-whitelist.json
JSON 
{
  "$schema": "https://raw.githubusercontent.com/rjohnn1/UPA-STANDARD/main/v1/lightweight-whitelist.json",
  "whitelist_version": "1.0.0",
  "mandate_currency": "USD",
  "last_authorized_by": "CFO-Principal",
  "approved_counterparties": {
    "VND-001": {
      "legal_name": "Cloud Analytics LLC",
      "legal_entity_identifier": "549300XXXXXXXXXXX123",
      "routing_bic": "CHESSUS33XXX",
      "max_single_transaction_limit": 5000.00,
      "requires_human_approval_above": 3000.00,
      "permitted_purpose_codes": ["COMP"],
      "permitted_jurisdictions": ["US"],
      "active": true
    }
  }
}
Field Type Required Description
approved_counterparties object Yes Map of vendor ID → counterparty configuration blocks enforced by the Fiduciary Translation Engine.
legal_name string Yes Full registered legal entity name. Must match government registry records.
routing_bic string · ISO 9362 Yes BIC/SWIFT routing code. Pattern: ^[A-Z]{6}[A-Z0-9]{2}([A-Z0-9]{3})?$
max_single_transaction_limit number > 0 Yes Per-transaction hard cap in mandate_currency. Fiduciary Layer enforces a hard block above this value.
permitted_purpose_codes string[] Yes ISO 20022 ExternalPurpose1Code values. e.g. COMP, TRAD, SALA, GDSV. Transactions with unmapped purpose codes are rejected.
legal_entity_identifier string · ISO 17442 — 20-character LEI. Required for RTGS and SWIFT CBPR+ cross-border transactions.
requires_human_approval_above number — Escalation threshold. Transactions above this value route to human review even if below the hard cap.
permitted_jurisdictions string[] · ISO 3166-1 — Alpha-2 country codes restricting which TargetJurisdictionString values are permitted for this vendor.
active boolean — If false, all payments to this vendor are blocked without deleting the record. Default: true.
Validate

Schema validation in 30 seconds

Validate any UPA-compliant whitelist file against this schema using standard JSON Schema tooling. The schema enforces field types, string patterns, required properties, and value constraints.

Compatible with any JSON Schema Draft 2020-12 validator. Python, JavaScript, Go, Java, Rust — all have production-grade implementations.

validate.py
Python 
import json, jsonschema, urllib.request

# Fetch the live raw schema from the production branch
url = "https://raw.githubusercontent.com/rjohnn1/UPA-STANDARD/main/v1/lightweight-whitelist.json"
schema = json.loads(
  urllib.request.urlopen(url).read()
)

# Load your local whitelist configuration
whitelist = json.load(open("whitelist.json"))

# Validate — raises ValidationError on non-compliance
jsonschema.validate(
  instance=whitelist,
  schema=schema
)

print("✓ Whitelist is UPA-1.2 compliant")
